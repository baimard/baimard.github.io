Serveur de gestion
==================

Présentation
------------

Toute la solution étant basée sur une base de données MariaDB, l'ensemble des données peut être administrées depuis un site internet dédié.

.. warning::

    * Ce site internet ne doit pas être exposé publiquement, il doit rester sur un réseau interne hautement sécurisé.
    * il peut être placé derrière un reverse proxy qui ajoutera une protection supplémentaire, même sur un réseau interne privé.

.. note::

    Vous pouvez accéder au site sur le lien par défaut : http://localhost:8083
    Le user par défaut est ``admin``, le mot de passe par défaut est : ``passwordtemporaire``

utilisation
-----------

La logique de l'utilisation de la solution est la suivante : 

* Création d'une clef dans le KSM
* Création d'un utilisateur
* Association d'une clef à un utilisateur
